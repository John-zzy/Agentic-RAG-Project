# 知识库数据处理层扩展方案

## Summary

目标是在现有“上传文件 -> 直接切块索引”的链路中，插入一个可人工确认的数据处理层，形成新的标准流程：

`上传文件 -> 弹窗预处理预览 -> 人工选择预设清洗规则 -> 按规则清洗 -> 按 chunk 策略切块 -> 向量化入库`

首版采用两个明确默认决策：

- 清洗规则采用“预设规则集”，不做字段级自定义规则编辑器。
- 交互采用“单次确认弹窗”，上传后立即进入预处理弹窗，在同一弹窗内完成规则选择、预览确认和开始入库。

## Key Changes

### 1. 平台层新增 Data Processing Layer

在 `backend/platform/knowledge` 下新增独立的 `processing` 子模块，职责放在 `documents` 之前，形成清晰分层：

- `loader` 继续负责从源文件读取原始记录。
- `processing` 负责标准化、清洗、预览、统计、溯源元数据生成。
- `chunker` 只接收“已清洗后的标准记录”进行切块。
- `service` 负责把整条链路编排起来。

处理流水线固定为：

1. 读取原始文件，产出原始 `DocumentRecord`
2. 根据文件类型做基础标准化
3. 应用用户选择的预设清洗规则
4. 生成处理结果统计和预览样本
5. 生成带溯源信息的清洗后记录
6. 按 `chunk_size/chunk_overlap` 切块
7. 写入向量库与文档主记录

首版预设规则固定为：

- `trim_whitespace`：去首尾空白、规范空行
- `drop_empty_records`：删除空内容记录
- `dedupe_records`：按规范化内容去重
- `strip_html_tags`：去 HTML 标签
- `remove_markdown_boilerplate`：去 Markdown 常见模板噪声
- `remove_url_lines`：删除纯链接/链接密集噪声行

规则适用范围固定为：

- `json` / `csv`：按记录级处理
- `txt` / `md`：按全文拆成单记录后处理
- `pdf` / `docx` / `xlsx`：首版继续允许上传，但不进入此处理链，也不能预处理/索引

### 2. 文档与版本模型补充处理配置和溯源摘要

扩展知识文档记录与版本摘要，新增以下逻辑字段：

- `source_type`：改为真实源类型，不再写死 `json`
- `processing_rules: list[str]`
- `processing_stats`
  - `raw_record_count`
  - `processed_record_count`
  - `removed_record_count`
  - `raw_char_count`
  - `processed_char_count`
- `provenance_enabled: bool`

扩展 chunk metadata，固定写入：

- `document_id`
- `document_version`
- `namespace`
- `source_type`
- `source_path`
- `source_record_id`
- `source_record_index`
- `chunk_id`
- `chunk_index`
- `applied_rules`
- `raw_content_hash`
- `processed_content_hash`
- `updated_at`
- `is_active`

这样后续无论是知识管理页查看详情，还是聊天引用扩展，都能追到“这个 chunk 来自哪个源文件、哪条记录、经过了哪些规则”。

### 3. API 扩展为“预览 + 确认处理”模式

保留现有 `/files/upload` 只做上传，不自动索引。

新增预处理预览接口：

- `POST /knowledge/documents/preprocess-preview`

请求体固定包含：

- `namespace`
- `source_path`
- `processing_rules`
- `chunk_size`
- `chunk_overlap`

返回固定包含：

- 文件基础信息与 `source_type`
- `supported_rules`
- `selected_rules`
- `processing_stats`
- `original_samples`
- `processed_samples`
- `can_index`
- `warnings`

扩展现有注册接口：

- `POST /knowledge/documents`

新增入参：

- `processing_rules: list[str]`

行为改为完整执行：

`load -> process -> chunk -> vectorize -> upsert record`

新增全文重处理接口：

- `POST /knowledge/documents/{document_id}/reprocess`

请求体包含：

- `processing_rules`
- `chunk_size`
- `chunk_overlap`
- `keep_version`

用途是已入库文档修改清洗规则后重建新版本。

保留现有：

- `POST /knowledge/documents/{document_id}/rechunk`

但语义收窄为“沿用当前版本处理规则，只改 chunk 参数重切块”。

扩展列表与详情响应：

- `/knowledge/documents/files` 中未入库但可处理文件的状态从 `unindexed` 改为 `awaiting_processing`
- `/knowledge/documents` 与详情接口返回处理规则和处理统计摘要

### 4. 前端知识管理页改为上传后弹出处理确认窗

知识管理页交互固定为：

1. 用户上传文件成功
2. 前端立即弹出“数据预处理”弹窗
3. 弹窗默认带上命名空间、默认 chunk 参数、空规则或推荐规则
4. 前端调用 `/knowledge/documents/preprocess-preview` 获取原始/清洗后预览
5. 用户勾选规则时，前端重新请求预览并刷新差异统计
6. 用户点击“确认处理并入库”
7. 前端调用 `/knowledge/documents` 完成实际入库
8. 成功后刷新文件列表和文档列表

对已索引文档，弹窗复用同一组件，但提交接口改为 `/reprocess`；仅修改 chunk 参数时可走 `/rechunk`。

弹窗必须展示的内容固定为：

- 源文件名、命名空间、文件类型
- 可勾选规则列表及规则说明
- 原始记录数 / 清洗后记录数 / 删除数
- 原始字符数 / 清洗后字符数
- 原始样本预览
- 清洗后样本预览
- chunk 参数输入框
- 不可索引时的明确原因提示

## Public APIs / Types

需要新增或调整的公开契约如下：

- 新增 `KnowledgeDocumentPreprocessPreviewRequest`
- 新增 `KnowledgeDocumentPreprocessPreviewResponse`
- 新增 `KnowledgeDocumentReprocessRequest`
- 扩展 `KnowledgeDocumentRegisterRequest.processing_rules`
- 扩展 `KnowledgeDocumentVersionResponse.processing_rules`
- 扩展 `KnowledgeDocumentDetailResponse.processing_stats`
- 扩展 `KnowledgeFileIndexSummaryResponse.status`
- 扩展内部 `DocumentRecord` 或新增 `ProcessedDocumentRecord`，用于承载清洗后内容和溯源字段

对外兼容策略固定为：

- 旧的 `POST /knowledge/documents` 请求若不传 `processing_rules`，后端按空规则列表处理
- 旧文档详情中如没有处理元数据，返回 `processing_rules=[]`、`processing_stats=null` 或零值对象
- `rechunk` 保持可用，不破坏现有调用方

## Test Plan

必须覆盖以下场景：

- 上传后请求预处理预览，返回规则列表、样本、统计值
- 不同规则组合能正确改变 `processed_record_count` 与样本内容
- `dedupe_records` 能稳定去重且不破坏 `source_record_id` 追踪
- `drop_empty_records`、`strip_html_tags`、`remove_url_lines` 对 `json/csv/txt/md` 行为符合预期
- 注册文档时会把 `processing_rules`、`processing_stats`、真实 `source_type` 写入文档记录
- 新生成 chunk metadata 包含溯源字段和规则字段
- `reprocess` 会生成新版本并替换激活版本
- `rechunk` 只改 chunk 参数，不改处理规则
- `/knowledge/documents/files` 对未处理文件返回 `awaiting_processing`
- 对 `pdf/docx/xlsx` 预览接口返回 `can_index=false` 与明确 warning
- 前端弹窗上传后自动打开；勾选规则后会刷新预览；确认后列表状态更新
- 兼容旧请求：不传 `processing_rules` 仍可成功入库

## Assumptions

- 首版只增强现有可索引类型：`json`、`txt`、`md`、`csv`
- 首版不引入异步任务系统，预览和处理都走同步 API
- 首版不做字段级规则编辑器，不做脚本型自定义规则
- 首版不改聊天接口返回结构，只先把引用溯源元数据完整写入知识侧
- 文档说明需同步更新 `README.md`、`docs/api-list.md`、`docs/data-model.md`，如知识管理页交互有明显变化，再补充相关页面说明
